# 外卖落下来啦！

手机优先的像素风外卖背包整理游戏网页版原型。

## 开始试玩

直接双击 `index.html`，或者在本目录启动本地服务：

```bash
python3 -m http.server 8000
```

然后打开 <http://localhost:8000>。

## 核心操作

- 手机游戏主界面固定为单屏：订单、天空、手持信息、背包和操作栏始终同时可见，不需要上下滚动。
- 外卖会沿三条轨道从天空持续坠落；按住包裹可以直接拖进背包，轻点接住仍然保留。
- 拖到绿色“+”松手即可吸附放下，大件会自动贴齐格线。
- 拖到绿色“合”上可以直接合成；背包里的两件食物也能互相拖动合成。
- 中间手持栏会持续用大字显示拿到的食物、鲜度和完整合成式。
- 交付后，中间手持栏会明确保留“交付了什么、收入多少”的记录。
- 红色“!”表示配方正确但成品没有连续空间，提示会说明需要的形状和被什么挡住。
- 拖动食物可以整理位置；点选后也可旋转或拿到手上。
- 接取和合成会让其他食物鲜度减 1，移动、旋转、提交订单不消耗鲜度。
- 鲜度归零会变成差评块，并继承原食物占据的完整形状。
- 原料或低阶食物不能直接交高阶订单；高阶成品可以替代低阶订单。
- 每班依次进入备餐期、午高峰和冲单期；掉落逐渐加快，冲单期交货有额外收入。
- 前三班渐进教学：奶茶合成、炸鸡旋转、综合配方依次开放。
- 首次进入采用5步互动教学：接第一杯、放入背包、接第二杯、直接合成、点击交单；漏接会自动补发，不会卡死。
- 配送站可以发送匿名试玩报告，只记录接取、放置、合成、失败与关卡结果，方便根据真实卡点调优。
- 掉落由订单缺口调度器驱动：会同时核算背包、手持和天空中的材料，优先补齐最前订单；漏掉必需材料后会重新补发。
- 连续漏接、无空间、错误合成或冷餐会触发动态放缓；连续成功后自动恢复正常节奏，无需玩家手动选择难度。
- 快餐篇现有10个正式班次，从奶茶教学逐步推进到L型汉堡、奶盖、2×2套餐、全鸡与最终综合考核。
- 每班拥有三星目标：完成配送、在目标时间内完成、全程无冷餐；记录永久最佳星级与最快时间。
- 配送地图会保存已解锁关卡、累计星数和总送达量；首次通关给完整奖励，重复通关只给少量奖励。
- 今日特送使用日期生成固定订单，同一天所有进入次数完全一致；首通奖励每天只能领取一次，但可以重复刷新星级与最快时间。

## 当前配方

```text
面包 + 牛肉饼 → 汉堡排（1×2）
汉堡排 + 芝士片 → 芝士汉堡（L型）
芝士汉堡 + 可乐 → 芝士汉堡套餐（2×2）

炸鸡 + 炸鸡 → 炸鸡桶（2×2）
炸鸡桶 + 炸鸡桶 → 全鸡（2×2）

奶茶 + 奶茶 → 大杯奶茶（1×1）
大杯奶茶 + 奶油 → 奶盖奶茶（1×2）
```

## 试玩版功能

- 鲜度、差评块、订单、高阶替代、升级与逐关难度
- 暖阳与赛博两套纯外观主题
- 本地进度保存和浏览器分享
- 桌面与手机响应式布局
